# MongoDB가 뭐냐?
- MongoDB는 BSON 또는 바이너리 JSON이라는 형식을 사용하여 JSON 문서를 저장한다.
- 비관계형 특성은 정형 애플리케이션 데이터뿐만 아니라 반정형 및 비정형 데이터를 저장할 수 있고 데이터베이스가 처리할 수 있음을 의미힌디.
## 장점
- 광범위한 용도로 사용되는 NoSQL이나, 주 사용처로는 콘텐츠 관리 시스템, 데이터 동기화가 빠르기에 특출난 장점을 가지고 있다.
- 유연성, 가용성, 확장성, 성능, 호환, 커뮤니티 지원 등 다양한 장점들이 존재한다.
## 단점
- 대용량 데이터 볼륨에서 유연성과 고성능이 필요한 애플리케이션의 경우 많은 잠재적인 단점이 존재한다.
- 복잡한 관계형성, 메모리 관리, 비용, 인덱스화 제한 사항, 스토리지 오버헤드 등이 발생할 수 있다.

## RDBMS vs NoSQL
- 데이터 모델, 스키마 확장성, 확장, 트랜잭션, 성능 등의 차이가 있는데 데이터 모델, 트랜잭션 빼고는 왠만하면 NoSQL이 좋은편이다.
